# EGoTouchRev 参数说明（模块级）

> 版本基线：当前 `D:\source\repos\EGoTouchRev-rebuild` 工作区代码。  
> 设备假设：固定单设备 `TX=40, RX=60`。  
> 目标：解释各算法模块参数的作用、默认值、范围和调参影响。  
> 策略：以“稳定性/可维护性/可调试性”为优先，不追求 OEM 每个状态位的 1:1 复刻。

## 0. 解算功能总览

### 0.1 触摸主链（Finger）
- `MasterFrameParser`：把硬件帧切成 `40x60` 主热力图与后缀状态区。
- `BaselineSubtraction -> CMF -> GridIIR`：完成去基线、去共模、时域稳定化。
- `FeatureExtractor`：从热力图生成候选触点（峰值、区域、面积、信号和、坐标）。
- `TouchTracker`：完成 ID 跟踪、TS/TE 连续性、报点门控、坐标平滑与事件输出。

### 0.2 手写笔主链（Stylus）
- `StylusProcessor`：解析 slave frame，恢复 TX1/TX2 9x9 block。
- `SolveStylusPoint`：双通道峰值/质心融合，生成 status 与频点切换请求。
- `ASA_MainProcess/HPP2/HPP3-lite`：做有效性判定、噪声后处理、recheck 与 touch suppress 触发。
- `Pressure/Tilt`：完成压力映射、频移后处理、倾角解算，最后打包手写笔包 `reportId=0x08`（13字节）。

### 0.3 触摸-手写笔互斥链（Interop）
- `TouchTracker` 内含 stylus suppression / StylusAFT-lite 分支。
- 输入依赖 `frame.stylus`（由 StylusProcessor 提供），输出作用在触摸 contacts 的保留/抑制。
- 该链路用于避免“笔写时手掌/指尖误触”污染触摸报点。

## 1. 通用规则

### 1.1 所有 Processor 的公共参数
- 配置键：`Enabled`
- 默认值：`true`（个别模块在类内部覆盖为 `false`）
- 作用：是否执行该模块 `Process()`

### 1.2 当前默认执行顺序（`RuntimeOrchestrator`）
1. 公共前处理：`MasterFrameParser -> BaselineSubtraction -> CMFProcessor -> GridIIRProcessor -> FeatureExtractor`
2. 手写笔分支：`StylusProcessor`（在公共前处理结果上单独执行）
3. 触摸分支：`TouchTracker`（注入当前帧 `stylus` 结果后执行）
4. 输出阶段：构建触摸 VHF 包（`0x20`）与手写笔 VHF 包（`0x0D`）

说明：
- Processor 注册顺序仍包含 `StylusProcessor` 与 `TouchTracker`，但运行时是“分支执行”，不是简单线性串行。
- `MasterFrameParser` 无可调参数（主要负责帧解析）。

### 1.3 运行模式开关（与算法相关）
- `MasterParserOnlyMode`：
  - 仅保留 `MasterFrameParser`，其余处理器临时禁用。
  - 用途：算法负载压测、原始热力图验证、排查后级回归。
- `AutoAfeFreqShiftSyncEnabled`：
  - 允许算法侧 `stylus.status bit0x40` 驱动 `switch_afe_mode(EnableFreqShift/ForceToFreqPoint)`。
  - 固定单设备场景下仅维护两频点索引（`idx0/idx1`）闭环。
- `VhfReportingEnabled`：
  - 控制触摸/手写笔包是否写入 VHF 设备；不影响核心解算流程本身。

## 2. BaselineSubtraction

模块作用：原始 AD 值减基线，得到差分信号中心化结果。

| 参数 | 配置键 | 默认值 | UI范围 | 作用与影响 |
|---|---|---:|---|---|
| Baseline Value | `BaselineValue` | `0x7FFE` (32766) | 输入任意 int；滑条 0~65535 | 基线越大，输出整体越偏负；越小，输出整体越偏正。建议先对齐空板均值再调后级阈值。 |

## 3. CMFProcessor（Common Mode Filter）

模块作用：去行/列共模漂移（充电器、LCD耦合等）。

| 参数 | 配置键 | 默认值 | UI范围 | 作用与影响 |
|---|---|---:|---|---|
| CMF Mode | `DimensionMode` | `RowWise(1)` | Disabled / Row / Column / Dual | 选择按行、按列或双维去共模。 |
| Exclusion Thrsh | `ExclusionThreshold` | 250 | 50~2000（log） | 高于阈值的点视为触摸，排除在噪声均值计算之外。过低会误排除背景，过高会把触摸带入均值。 |
| Max Correction | `MaxCorrection` | 500 | 10~2000（log） | 限制每行/列的校正上限，防止过校正。 |

## 4. GridIIRProcessor

模块作用：网格级时域低通，抑制帧间抖动。

| 参数 | 配置键 | 默认值 | UI范围 | 作用与影响 |
|---|---|---:|---|---|
| IIR Weight | `IIRWeight` | 120 | 1~256 | 当前帧权重。越小越平滑（也更“粘”）；越大越跟手。 |
| Adaptive Threshold | `AdaptiveThreshold` | 15 | 0~500 | 当 `|current-history|` 超过阈值时旁路 IIR。阈值越小，越容易旁路，动态响应更快。 |

| TE Recover Distance | `TERecoverDistance` | 2.8 | 0.5~10.0 | 回收距离阈值（热力图坐标系）。值越小更保守，值越大更激进。 |
| TE Recover Direction Constraint | `TERecoverDirectionConstraint` | true | 开关 | 回收时要求“当前位移方向”与“释放前速度方向”一致，可抑制串 ID 误回收。 |
| TE Min Release Speed | `TERecoverMinReleaseSpeed` | 0.4 | 0.05~4.0 | 仅当释放前速度超过该阈值时启用方向约束；低速/静止轨迹不强约束。 |
| TE Min Recover Cos | `TERecoverMinCosine` | 0.15 | -1.0~1.0 | 方向一致性阈值（余弦）。越大越严格，越小越宽松。 |

说明：
- 恢复逻辑仅优先回收“此前已上报”的轨迹，避免将纯噪点轨迹复活。
- 回收命中后事件按 `Move` 输出，避免重复 `Down`。
- 若方向约束开启，回收需同时满足距离与方向条件；能显著降低相邻手指串 ID。

#### 6.7.1 TS/TE 调参速查

> 适用版本：`EGoTouchRev-rebuild` 当前实现（`Touch Tracker (IDT/TS/TE-lite)` + TS/TE Continuity）。
> 目标：快速定位 `Down/Move/Up` 抖动、误抬起、ID 跳变，并给出可操作的调参顺序。

**典型问题：**
1. 边缘滑动时出现 `Up -> Down` 快速闪烁。
2. 快速移动时偶发 ID 断裂，重新分配了新 ID。
3. 抬手后 `Up` 偶尔太慢，感觉"拖尾"。
4. 静止时点位不抖，但慢速移动容易被抑制不报。

**推荐调参顺序：**
1. 先固定前级：`FeatureExtractor` 阈值不动，先只调 `TouchTracker`。
2. 先调 `TELostGraceFrames`：从 `1` 开始，问题重则加到 `2`。
3. 再调 `TERecoverDistance`：从 `2.8` 往下减到 `2.2` 看误回收是否下降。
4. 再调方向约束：先开 `TERecoverDirectionConstraint=true`，`TERecoverMinCosine` 从 `0.15` 往上调。
5. 再调 `TERecoverWindowFrames`：一般不超过 `3`，避免远距离误复活。
6. 最后才动 `StartGate` 和 `Report*Suppress`，避免把问题掩盖掉。

**场景化建议：**

- **边缘抖动明显**（最常见）：`TELostGraceFrames` 1→2, `TERecoverDistance` 2.6~3.0, `TERecoverMinCosine` 0.10~0.25。
- **快速滑动时 ID 断裂**：`TERecoverWindowFrames` 2→3, `TERecoverDistance` 2.8→3.2, `TERecoverMinCosine` 0.05~0.20；若仍断裂可小幅提高 `MaxTrackDistance`。
- **抬手拖尾（Up 太晚）**：`TELostGraceFrames` 2→1→0, `TERecoverWindowFrames` 2→1, `TERecoverDistance` 降低至 2.2。
- **误回收（串 ID）**：`TERecoverDistance` 降至 2.0, `TERecoverWindowFrames` 降至 1, `TERecoverMinCosine` 提至 0.30；必要时 `TERecoverEnabled=false` 做 A/B 对比。

**推荐预设：**

默认稳健：`TELostGraceFrames=1, TERecoverWindowFrames=2, TERecoverDistance=2.8, TERecoverMinReleaseSpeed=0.4, TERecoverMinCosine=0.15`

边缘抗抖优先：`TELostGraceFrames=2, TERecoverDistance=3.0, TERecoverMinReleaseSpeed=0.35, TERecoverMinCosine=0.10`

低延迟优先：`TELostGraceFrames=0, TERecoverWindowFrames=1, TERecoverDistance=2.2, TERecoverMinReleaseSpeed=0.5, TERecoverMinCosine=0.05`

**验证建议：**
1. 每次只改 1~2 个参数。
2. 用同一段回放做 A/B（建议 3000 帧离线压测 + UI 实时观察）。
3. 重点看三项：`Up` 是否稳定且不拖尾、ID 是否连续、边缘是否出现 `Up->Down` 闪烁。

### 6.9 Stylus Processor（新增）
功能流程（按帧）：
- `ParseSlaveWords`：从 master+slave 缓冲中定位 166 words，并做 checksum/header 评分。
- `SolveStylusPoint`：TX1/TX2 双通道峰值+质心融合，生成 `point/status/freq`。
- `RunAsaMainProcess + RunStylusRecheck`：执行 HPP2/HPP3-lite 判定、噪声门控、重叠/recheck。
- `SolveStylusPressure/SolveStylusTilt`：输出 pressure、tilt、report 坐标，并写入 packet。

说明：
- 当前实现按原厂链路拆分为 `slave解析 -> 9x9映射 -> 双通道解算 -> 状态/频切 -> packet打包`。
- 手写笔解算新增：`report坐标`、`raw/mapped压力`、`pre-tilt/tilt/tilt方位角`，均可在 UI 和 CSV 导出查看。
- `NoPressInk` 表示“笔在 in-range 且状态有效，但当前压力为0”。原厂中该状态通常用于放宽/跳过部分 recheck 失败，避免悬笔、轻触和切频瞬态导致误退笔。
- `status` 基值使用 `0x06/0x12`，并叠加 `0x20`（不稳定）与 `0x40`（切频请求）。
- `SolveStylusPressure/SolveStylusTilt` 已移动到状态位生成之后执行，`in-range` 与 `freq-shift` 分支可直接读取当前帧 `status`。
- 该模块不参与手指点提取，仅提供手写笔结果与可直接用于后续 HID 注入的 packet。

## 7. 可选模块（当前默认管线未启用）

### 7.1 GaussianFilter
- 默认 `Enabled=true`（若加入管线）
- 参数：`Center Kernel Weight`（默认 20，范围 1~30）
- 说明：中心权重越大，模糊越弱。
- 注意：该模块未实现 `SaveConfig/LoadConfig`，参数不会持久化（仅会话有效）。

### 7.2 SpatialSharpenFilter
- 默认 `Enabled=false`
- 参数：`Sharpening Strength`（配置键 `Strength`，默认 1.0，范围 0.1~5.0）
- 说明：增强局部梯度，利于近邻峰分离。

### 7.3 SignalConditioningFilter
- 默认 `Enabled=true`（若加入管线）
- 参数：
  - `IIR Alpha`（键 `Alpha`，默认 700，范围 100~1000）
  - `Noise Cut-off Floor`（键 `NoiseFloor`，默认 80，范围 0~500）
- 说明：先时域平滑，再做噪声地板截断。

### 7.4 SignalSegmenter
- 默认 `Enabled=false`
- 参数：
  - `BaseThreshold`（50）
  - `HighSignalRatio`（0.45）
  - `MediumSignalRatio`（0.4，当前 UI 未直接暴露）
  - `DampDist1`（0.8）
  - `DampDistSqrt2`（0.5）
  - `DampDist2`（0.2）
- 说明：用于 Phase-1 分段与峰检测验证。

### 7.5 CentroidExtractor
- 默认 `Enabled=true`（若加入管线）
- 参数（仅 UI，未持久化）：
  - 算法选择 `m_algorithm`：`Native PCA` / `2D Paraboloid`
  - `Peak Detection Threshold`（默认 80，范围 50~2000）
  - `m_minPeakDist`（默认 4.0，当前未在 UI 暴露）
- 说明：层次 PCA-KMeans + 抛物面细化质心。

## 8. 调参建议（实战顺序）

1. 先定前处理稳定性：`Baseline -> CMF -> GridIIR`。  
2. 再定触点提取：`FeatureExtractor` 的 `PeakThreshold / ZoneThreshRatio / PreFilter`。  
3. 最后定事件体验：`TouchTracker` 的 `StartGate / Debounce / ReportFilter / CoordinateFilter / TS/TE Continuity`。  
4. 若追求“更原厂手感”，优先动 `TouchTracker` 的报点抑制与回放参数，而不是先放宽前级阈值。
5. 若出现“边缘快速滑动时 Up/Down 抖动”，优先调 `TELostGraceFrames` 与 `TERecoverDistance`，再考虑放宽前级阈值。
