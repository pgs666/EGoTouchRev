# 设计决策与差异（非 1:1 还原取舍 + 后续优化方向）

> 更新日期：2026-03-19
> 目标：集中记录本项目相对于 OEM 原厂实现的有意取舍、差异说明，以及后续优化方向。

## 1. 非 1:1 还原的有意取舍

为了优化/重构目标，当前有意不做以下 OEM 细节复刻：

### 1.1 写门控状态机

- 未复刻 OEM 的 `DAT_1800c73ac` 写入门控状态机。
  - `DAT_1800c73ac` 为 OEM 写入门控位（`VhfWriteFingerFile/VHF_InjectPenData` 均会检查），当前工程仅保留 UI 开关门控，尚未找到该位的完整状态机来源。
  - 本项目使用 `VhfReportingEnabled` 作为应用层输出门控。

### 1.2 全局状态与命名管道

- 未完整镜像 OEM 所有 pen 全局状态位/通知管道。
  - 仅保留当前算法与 VHF 回报实际需要的事件与状态。

### 1.3 频率跳变事件处理

- `PEN_FREQ_JUMP(0x79)` 事件不直接驱动 Himax 频点命令。
  - 频点切换由算法侧 `status bit0x40` + `AutoAfeFreqShiftSync` 闭环控制。

## 2. 仍待确认的 OEM 细节

- `DAT_1800c73ac` 的完整状态机来源与所有写入路径。
- USB 头部各字段完整语义（尤其第7字节）在不同命令下存在特例，建议后续结合抓包确认。
- 120/240Hz 与具体 `freq_index` 的精确映射需结合项目参数或运行时日志进一步落地。

## 3. 后续优化方向

1. 在 `PenSession` 增加超时/断链重连策略（指数退避 + 自动重建会话）。
2. 增加事件统计与丢包诊断导出（用于线上问题回放）。
3. 将 ACK 失败从日志告警升级为可观测计数（UI/CSV）。
4. 在不改 API 的前提下，逐步补齐更多事件语义解析。
