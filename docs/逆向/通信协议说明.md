# 通信协议说明（BT MCU USB 协议 + VHF 回报）

> 更新日期：2026-03-19
> 目标：统一记录本项目中 BT MCU 的 USB 私有协议与 VHF 回报链路的包格式、事件、ACK 机制及当前实现状态。

## 1. 模块分层

- `Device/include/himax` + `Device/source/himax`
  - Himax 芯片寄存器、命令、采集与 AFE 模式切换。
- `Device/include/btmcu` + `Device/source/btmcu`
  - BT MCU 私有 USB 协议（包格式、编解码、会话、线程客户端、Win32 传输）。
- 兼容头：
  - `Device/include/HimaxChip.h`
  - `Device/include/HimaxProtocol.h`
  - `Device/include/HimaxRegisters.h`

### 1.1 btmcu 子模块职责

- `PenUsbTypes.*`：命令 ID、事件码、协议头/包结构、回调类型。
- `PenUsbFrame.*`：8 字节头与 payload 编解码。
- `PenUsbCodec.*`：命令构包、原始包解析、事件包识别。
- `PenUsbTransport.h` + `win32/PenUsbTransportWin32.cpp`：Win32 句柄读写。
- `PenSession.*`：会话状态、收发统计、事件回调分发。
- `PenUsbClient.*`：后台轮询线程，`QueryBootstrapInfo / SendEventAck`。
- `PenCommandApi.*`：供上层调用的稳定接口。

## 2. USB 私有协议包格式

- 发送包头固定 8 字节：
  - `[reportId, direction, protocol, reserved, cmd_lo, cmd_hi, tag, option]`
- 当前固定常量：
  - `reportId=0x07`
  - `protocol=0x02`
  - `transportTag=0x11`
- 常用命令：
  - `0x7101` 查询状态
  - `0x7701` 查询信息
  - `0x8001` 事件 ACK

## 3. 事件分发与 ACK

### 3.1 事件分发表（上行 MCU->Service）

当前 `PenUsbCodec::IsEventPacket` 支持：
- `0x2F`（`PenCurrentFunc`）
- `0x70~0x7F`（Pen 状态事件区间）

### 3.2 ACK 映射

Runtime 已实现 ACK 映射并回发 `0x8001`：

| 事件码 | ACK code | 语义 |
|---:|---:|---|
| `0x70` | 0 | `PEN_AC_STATUS` |
| `0x71` | 1 | `PEN_CONN_STATUS` |
| `0x72` | 2 | `PEN_CUR_STATUS` |
| `0x73` | 13 | `PEN_TYPE_INFO` |
| `0x74` | 3 | `PEN_ROATE_ANGLE` |
| `0x75` | 4 | `PEN_TOUCH_MODE` |
| `0x76` | 5 | `PEN_GLOBAL_PREVENT_MODE` |
| `0x77` | 6 | — |
| `0x78` | 7 | `PEN_HOLSTER` |
| `0x79` | 8 | `PEN_FREQ_JUMP` |
| `0x7B` | 10 | — |
| `0x7C` | 12 | `PEN_GLOBAL_ANNOTATION` |
| `0x7F` | 9 | `ERASER_TOGGLE` |
| `0x2F` | 11 | `PenCurrentFunc` |

## 4. VHF 触摸包（0x20）布局

- 包头：`byte[0] = 0x01`
- 点槽：每点 6 字节，共 5 点/包
  - `byte[1 + i*6 + 0]`：点状态（OEM链路中来自点结构 offset `+0x00`）
  - `byte[1 + i*6 + 1]`：点 ID（OEM链路中来自点结构 offset `+0x10`）
  - `byte[1 + i*6 + 2..3]`：X（LE）
  - `byte[1 + i*6 + 4..5]`：Y（LE）
- `byte[31]`：总触点数（0~10，写在第 1 包）
- 坐标变换：
  - `x_vhf = 16000 - rawX * 10`
  - `y_vhf = rawY * 10`
- 触点超过 5 个时发送第 2 包（同样 0x20）。

## 5. VHF 手写笔包（0x0D）布局

- 包头：
  - `byte[0] = 0x08`
  - `byte[2] = 0x80`
- `byte[1]`：状态位（bit0 + bit5/bit6）
- `byte[3..4]`：X（LE）
- `byte[5..6]`：Y（LE）
- `byte[7..8]`：压力（LE）
- `byte[9..10]`：tiltX（LE, int16）
- `byte[11..12]`：tiltY（LE, int16）

### 5.1 Eraser 后处理（与 BT MCU 联动）

Runtime 监听 `ERASER_TOGGLE(0x7F)` 并缓存状态。在发送 VHF 笔包前，对 `byte[1]` 做后处理（对齐 OEM `VHF_InjectPenData`）：
- `eraser_state == 1`：`byte1 = (byte1 & 0xFE) | 0x0C`
- 其他：`byte1 = byte1 & 0xF3`

## 6. 当前代码落地状态

### 6.1 Runtime 接入

- `App::RuntimeOrchestrator` 在 `Start()` 时执行 `InitializePenBridge()`：
  - 通过 GUID `{DD0EBEDB-F1D6-4CFA-ACCA-71E66D3178CA}` 枚举并打开 BT MCU 设备。
  - 启动 `PenCommandApi` 轮询线程。
  - 启动后发送 `QueryBootstrapInfo`（`0x7101` + `0x7701`）。

### 6.2 固件模式切换接口

- 统一命令入口：
  - `Device/include/HimaxChip.h`
  - `Chip::switch_afe_mode(AFE_Command cmd, uint8_t param)`
- `AFE_Command` 覆盖：
  - `ClearStatus` / `EnableFreqShift` / `DisableFreqShift` / `StartCalibration`
  - `EnterIdle` / `ForceExitIdle` / `ForceToFreqPoint` / `ForceToScanRate`
- 上层入口：
  - `App/include/RuntimeOrchestrator.h`
  - `RuntimeOrchestrator::SwitchAfeMode(Himax::AFE_Command cmd, uint8_t param)`

参数透传已修复并确认生效：
- `thp_afe_enter_idle(param)` -> `cmd 0x0A`
- `thp_afe_start_calibration(param)` -> `cmd 0x01`
- `thp_afe_force_to_scan_rate(rate_idx)` -> `cmd 0x0E`

### 6.3 自动频移闭环（单设备优化版）

- 在 `RuntimeOrchestrator::HandleAutoAfeFreqShiftSync` 中实现：
  - 自动发送一次 `EnableFreqShift(0x0D)`。
  - 从 `stylus.tx1Freq/tx2Freq` 学习 `idx0` 频点对（单设备场景）。
  - 当 `stylus.status & 0x40` 置位时，根据 `nextTx1/nextTx2` 判定目标 `idx` 并下发 `ForceToFreqPoint(0x0C)`。
  - 内置冷却时间，避免重复强制切点。
- 这条闭环以"当前设备稳定运行"为目标，不做 OEM 多机型频点策略泛化。

### 6.4 VHF 设备输出链路

- 新增 VHF 设备输出链路：
  - 使用 GUID 枚举设备接口并 `CreateFileW` 打开。
  - 触摸包按 32 字节写入，手写笔包按 13 字节写入。
  - 写失败后自动 `Close -> Sleep(200ms) -> Reopen`。

### 6.5 触摸打包修正

- 前两字节语义调整为 `state/id`。
- 去除按 ID 重新排序，保持跟随算法输出顺序。

### 6.6 调试与导出

- UI 增加 VHF 开关和设备打开状态显示。
- Touch Debug 增加 `TouchPacket[0/1]` 十六进制显示。
- DVR/CSV 导出增加触摸包十六进制内容。
