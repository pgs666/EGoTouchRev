# EGoTouchRev 项目总结

> 更新日期：2026-03-19
> 代码基线：`D:\source\repos\EGoTouchRev-rebuild` 工作区

## 1. 项目概述

EGoTouchRev 是一个触控面板（Touch Panel）驱动算法的逆向重建项目。基于对原厂 OEM 驱动（`THP_Service.dll`、`himax_thp_drv.dll`、`TSACore.dll`）的 Ghidra 静态逆向分析，独立重建了从硬件数据采集到 HID 回报的完整触控处理链路。

**核心目标：**
- 在 Windows ARM64 平台上独立运行触控算法，脱离 OEM 闭源驱动
- 支持多点触摸（最多 10 点）与手写笔（含压力、倾角）
- 提供实时调参 UI，支持算法可视化与离线回放调试
- 通过 VHF（Virtual HID Framework）向操作系统回报触摸/手写笔事件

**技术栈：**
- 语言：C++23
- 构建：CMake 3.20+
- 平台：Windows ARM64（支持 NEON SIMD）
- UI：ImGui (DX11 backend)
- 日志：spdlog 1.17.0

## 2. 项目架构

项目采用分层模块化架构，每个模块为独立的 CMake 静态库（App 为可执行文件）：

```
EGoTouchRev-rebuild/
├── Common/        静态库 - 公共工具、ImGui、spdlog
├── Device/        静态库 - 硬件抽象（Himax 芯片、BT MCU、VHF 设备）
├── Engine/        静态库 - 触控算法引擎（信号处理、特征提取、跟踪）
├── Host/          静态库 - 系统集成（SPI 通信、系统状态监控）
├── Control/       静态库 - 用户态服务控制面（命名管道 IPC）
├── App/           可执行文件 - 主程序（RuntimeOrchestrator + ImGui UI）
├── BTMCUTool/     可执行文件 - BT MCU 协议验证工具
└── docs/          技术文档
```

**依赖关系：**
```
App → Device, Engine, Host, Common, D3D11, setupapi
Device → Common
Engine → Common
Host → Common, advapi32
Control → Host, Common
BTMCUTool → Device, Common, D3D11, setupapi
```

## 3. 各模块详解

### 3.1 Common（公共基础库）

提供项目共享的基础设施：
- ImGui docking 分支（UI 框架）
- spdlog 日志库集成
- 公共头文件与工具函数

### 3.2 Device（硬件抽象层）

负责所有硬件设备的通信与抽象：

**Himax 芯片子模块（`himax/`）：**
- `HimaxChip`：芯片命令接口，统一的 `switch_afe_mode(AFE_Command, param)` 入口
- `HimaxProtocol`：16 字节命令帧构建、校验计算、命令槽管理
- `HimaxRegisters`：寄存器地址常量
- 支持的 AFE 命令：EnableFreqShift / DisableFreqShift / ClearStatus / StartCalibration / EnterIdle / ForceExitIdle / ForceToFreqPoint / ForceToScanRate

**BT MCU 子模块（`btmcu/`）：**
- `PenUsbTypes`：命令 ID、事件码、协议头/包结构定义
- `PenUsbFrame`：8 字节头与 payload 编解码
- `PenUsbCodec`：命令构包、原始包解析、事件包识别
- `PenUsbTransport`：Win32 USB 句柄读写（平台相关层）
- `PenSession`：会话状态管理、收发统计、事件回调分发
- `PenUsbClient`：后台轮询线程，自动 Bootstrap 查询与 ACK
- `PenCommandApi`：供上层调用的稳定 API 接口

**Runtime 子模块（`runtime/`）：**
- `DeviceRuntime`：设备运行时，包含 Worker 线程状态机

### 3.3 Engine（算法引擎）

实现完整的触控信号处理管线：

| 处理器 | 职责 |
|---|---|
| `MasterFrameParser` | 硬件帧解析，切分 40×60 主热力图与状态区 |
| `BaselineSubtraction` | 原始 AD 值减基线（默认 `0x7FFE`） |
| `CMFProcessor` | 行/列共模滤波（去充电器/LCD 耦合噪声） |
| `GridIIRProcessor` | 网格级时域低通（帧间抖动抑制） |
| `FeatureExtractor` | 峰值检测、区域分割、预过滤、质心计算 |
| `StylusProcessor` | Slave frame 解析、双通道解算、压力/倾角/频点 |
| `TouchTracker` | ID 跟踪、TS/TE 连续性、报点门控、坐标滤波 |
| `GaussianFilter` | 可选空间平滑 |
| `SpatialSharpenFilter` | 可选空间锐化 |
| `SignalConditioningFilter` | 可选时域平滑+噪声截断 |

**执行顺序：**
1. 公共前处理：`MasterFrameParser → BaselineSubtraction → CMF → GridIIR → FeatureExtractor`
2. 手写笔分支：`StylusProcessor`
3. 触摸分支：`TouchTracker`（注入 stylus 结果后执行）
4. 输出：VHF 触摸包（32 字节）+ 手写笔包（13 字节）

### 3.4 Host（系统集成层）

- SPI 通信接口（与 Himax 芯片的底层数据传输）
- 系统状态监控（命名事件机制）
- 依赖 `advapi32`（Windows 服务/注册表 API）

### 3.5 Control（服务控制面）

- `ControlRuntime`：用户态服务运行时
- `ControlPipeServer`/`ControlPipeClient`：命名管道 IPC
- `ControlServiceMain`：独立服务进程入口
- `ControlServiceTestTool`：服务控制测试工具

### 3.6 App（主程序）

- `RuntimeOrchestrator`：核心编排器，协调 Device/Engine/Host 的启动与运行
- ImGui DX11 界面：实时热力图、参数调节、调试面板
- VHF 输出管理：触摸/手写笔包的构建与写入
- 自动频移闭环（`HandleAutoAfeFreqShiftSync`）
- BT MCU 事件桥接（Eraser toggle → VHF 笔包后处理）

### 3.7 BTMCUTool（协议验证工具）

独立的 ImGui 应用，用于验证 BT MCU USB 私有协议的收发、事件识别与 ACK 机制。

- Windows 11 ARM64

## 7. 设计理念

本项目以**稳定性、可维护性、可调试性**为优先，不追求 OEM 每个状态位的 1:1 复刻。关键设计决策包括：

- 使用应用层 `VhfReportingEnabled` 替代 OEM 的复杂写门控状态机
- 频率切换统一由算法状态位闭环驱动，不依赖 `PEN_FREQ_JUMP` 事件
- 仅保留算法与 VHF 实际依赖的关键事件与状态，不镜像全部 OEM 全局变量
